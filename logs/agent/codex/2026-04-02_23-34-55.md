- 実装理由または判断根拠:
  - 原因分析として、金額が表示される主要カードだけが `font-display` である `Syne` を使っており、一覧や他 UI の数値フォントと混在していた
  - `Syne` は見出し向けの装飾フォントで、数字や記号のベースライン・字面が本文フォントより不安定になりやすく、`¥` や `+` `-` を含む文字列で上下ぶれが見えやすい構成だった
  - さらに通貨記号を前置する `¥` 形式は、数字本体と記号の見た目の重心差が強く、視覚的なぶれを強調していた
  - そのため、根本対応として「金額を見出しフォントから外す」「数字用のフォント指定と numeric variant を共通化する」「通貨を後置して数字中心の見え方へ寄せる」を採用した
- 実行した確認内容:
