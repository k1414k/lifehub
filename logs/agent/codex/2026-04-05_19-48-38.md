  - `typescript` `@types/node` `@types/react` `@types/react-dom` を `devDependencies` から `dependencies` に移動
  - `package-lock.json` を更新し、production install 時にも上記 build 依存が解決される状態に修正
- 実装理由または判断根拠:
  - 既存の `globals.css` は Tailwind v3 の `@tailwind base;` 形式で、ビルド時に PostCSS/Tailwind が必須
  - ローカルの通常ビルドは成功していた一方、`npm ls --omit=dev tailwindcss postcss autoprefixer --depth=0` では対象パッケージが空になっており、production install ベースの環境では CSS 前処理が欠落することを確認
  - 追加で `npm ci --omit=dev` 相当のクリーン検証を行ったところ、Tailwind 系を移した後は CSS パースエラーは再現せず、今度は TypeScript 系不足に起因する `@/*` 解決エラーが見えた
  - このため「本番ビルドで必要な依存は dependencies に寄せる」方針で、Tailwind/PostCSS だけでなく TypeScript 系も production install に残す構成へ修正した
  - デプロイスクリプト本体がリポジトリ外にあり、EC2 側で `--omit=dev` 相当の install をしている可能性を安全側で吸収する目的
- 実行した確認内容:

  - `cd frontend && npm run build`
